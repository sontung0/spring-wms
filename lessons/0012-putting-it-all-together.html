The Grand Finale

You've learned the core components of Spring Integration: Channels, Endpoints, Gateways, Routers, Service Activators, Filters, Transformers, Splitters, Aggregators, and Channel Adapters. Now it's time to see how they all fit together to solve a real-world problem.

A Complete Integration Flow

Let's design a comprehensive flow for spring-wms. Here is the scenario:

  1. Inbound: We receive a batch of inventory updates via an HTTP POST request (JSON array).
  2. Split: We split the batch into individual update messages.
  3. Filter: We filter out any updates with a quantity of 0 (we don't care about empty stock).
  4. Transform: We transform the JSON payload into a Java InventoryUpdate object.
  5. Route: We route the updates based on the warehouse location (e.g., "North" vs. "South").
  6. Process: We process the updates using a Service Activator (saving to the database).
  7. Aggregate: We aggregate the results back together.
  8. Outbound: We write a summary report to a file.
@Bean
public IntegrationFlow inventoryUpdateFlow() {
    return IntegrationFlow.from(Http.inboundGateway("/api/inventory")
                                    .requestMapping(m -> m.methods(HttpMethod.POST)))
        
        // 1. Split the JSON array into individual JSON objects
        .split() 
        
        // 2. Filter out items with quantity 0 (assuming JSON string manipulation or JsonPath)
        .filter(String.class, json -> !json.contains("\"quantity\":0")) 
        
        // 3. Transform JSON to Java Object
        .transform(Transformers.fromJson(InventoryUpdate.class)) 
        
        // 4. Route based on warehouse location
        .<InventoryUpdate, String>route(update -> update.getWarehouse(), mapping -> mapping
            .subFlowMapping("North", sf -> sf.handle(inventoryService, "updateNorth"))
            .subFlowMapping("South", sf -> sf.handle(inventoryService, "updateSouth"))
        )
        
        // 5. Aggregate the results
        .aggregate() 
        
        // 6. Transform the aggregated list into a String report
        .transform(list -> "Processed " + list.size() + " updates.") 
        
        // 7. Write the report to a file
        .handle(Files.outboundAdapter(new File("/tmp/wms/reports"))
                     .fileExistsMode(FileExistsMode.APPEND)
                     .appendNewLine(true))
        .get();
}

Note: In a real application, you'd likely use an errorChannel to handle failures during this complex process, as we learned in Lesson 8.
