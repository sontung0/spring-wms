Lesson 4: Routing Messages

Awesome job getting the first flow running! So far, our flow is completely linear: Message goes in, gets transformed, gets printed.

But in a real system (like a Warehouse Management System), you often need to make decisions. For example: If an order is for "Electronics", send it to warehouse A; if it's for "Clothing", send it to warehouse B.

This is where the Router comes in.

The Message Router

A Router consumes a message from an input channel and determines which output channel(s) it should be sent to. It doesn't modify the message; it just directs traffic.

Routing with Java DSL

The DSL makes routing very intuitive using the route() method. You usually route based on the payload (the data) or the message headers.

import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.integration.dsl.IntegrationFlow;

@Configuration
public class RoutingFlowConfig {

    @Bean
    public IntegrationFlow orderRoutingFlow() {
        return IntegrationFlow.from("orderChannel")
            // Route based on the payload (assuming payload is a String like "ELECTRONICS" or "CLOTHING")
            .<String, String>route(payload -> payload, mapping -> mapping
                .channelMapping("ELECTRONICS", "electronicsChannel")
                .channelMapping("CLOTHING", "clothingChannel")
                // What happens if we get "FOOD"? We need a default!
                .defaultOutputToParentFlow() 
            )
            .get();
    }
    
    // We can define separate sub-flows for the routed channels!
    @Bean
    public IntegrationFlow electronicsFlow() {
        return IntegrationFlow.from("electronicsChannel")
            .handle(m -> System.out.println("Processing Electronics Order: " + m.getPayload()))
            .get();
    }

    @Bean
    public IntegrationFlow clothingFlow() {
        return IntegrationFlow.from("clothingChannel")
            .handle(m -> System.out.println("Processing Clothing Order: " + m.getPayload()))
            .get();
    }
}

Anatomy of the route() definition

  1. .<String, String>route(payload -> payload...): We are saying the payload is a String, and the value we want to use to make our routing decision is the payload itself. If you had an `Order` object, it might be `order -> order.getCategory()`.
  2. mapping -> mapping.channelMapping(...): This is where we map the result of our routing function to a specific channel name.
