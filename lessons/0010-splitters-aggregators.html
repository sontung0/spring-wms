Lesson 10: Splitters and Aggregators

Mission: Master Spring Integration for spring-wms

Divide and Conquer

Often in integration, you receive a "batch" of data—like a file containing 100 inventory updates, or a JSON array of new orders. Processing the entire batch as a single unit can be inefficient or difficult, especially if some items fail while others succeed.

The Splitter pattern breaks a single message containing a collection into multiple individual messages. The Aggregator pattern does the exact opposite: it collects multiple related messages and combines them back into a single message.

Using Splitters and Aggregators in Java DSL

1. The Splitter (.split())

The .split() method is incredibly smart. If your message payload is a List, an array, or any Iterable, calling .split() without arguments will automatically create a new message for every item in that collection.

@Bean
public IntegrationFlow processBatchFlow() {
    return IntegrationFlow.from("batchInputChannel")
        // Payload is List<Order>. Splitter turns this into many Message<Order>
        .split() 
        // This handle method is now called once for EACH order
        .handle(orderService, "processSingleOrder") 
        .get();
}

Magic behind the scenes: When a Splitter creates these individual messages, it automatically adds special headers to them: a correlationId (to group them together), a sequenceSize (total items), and a sequenceNumber (the item's index). These headers are crucial for the Aggregator.

2. The Aggregator (.aggregate())

After splitting and processing items individually, you might want to send a single summary report. The .aggregate() method collects messages until a group is complete.

By default, it uses the headers added by the Splitter (correlationId, sequenceSize) to know exactly which messages belong together and when the group is fully assembled.

@Bean
public IntegrationFlow splitProcessAggregateFlow() {
    return IntegrationFlow.from("batchInputChannel")
        .split() // 1 Message (List of 5) -> 5 Messages
        
        // Process each item (could be done in parallel using an ExecutorChannel!)
        .transform(order -> enrichOrderData(order)) 
        
        .aggregate() // 5 Messages -> 1 Message (List of 5 enriched orders)
        
        // Now we have a single List again, send the summary
        .handle(reportService, "sendSummaryReport") 
        .get();
}
