Lesson 11: Channel Adapters

Mission: Master Spring Integration for spring-wms

Connecting to the Outside World

So far, we've been passing messages around inside our own application. But the true power of Spring Integration is connecting to external systems: reading files from a disk, listening to a RabbitMQ queue, polling a database, or sending an email.

We do this using Channel Adapters. They act as the bridge between the external world and your internal Spring Integration channels.

Inbound vs. Outbound Adapters

1. Inbound Channel Adapters (The Source)

Inbound adapters bring data into your application. They are the starting point of an IntegrationFlow. There are two main types:

  • Message-Driven (Event-Driven): The external system pushes data to us (e.g., listening to an AMQP queue or an HTTP endpoint).
  • Polling: We actively check the external system on a schedule (e.g., checking a directory for new files every 5 seconds).
import org.springframework.integration.file.dsl.Files;
import java.io.File;

@Bean
public IntegrationFlow fileReadingFlow() {
    // Polling Inbound Adapter: Checks the directory every 1 second
    return IntegrationFlow.from(Files.inboundAdapter(new File("/tmp/wms/inbound"))
                                     .patternFilter("*.csv"),
                                e -> e.poller(Pollers.fixedDelay(1000)))
        .transform(Transformers.fileToString())
        .handle(System.out::println)
        .get();
}

2. Outbound Channel Adapters (The Destination)

Outbound adapters send data out of your application. They are usually the end point of an IntegrationFlow. They take a Spring Integration Message and write it to a file, send it to a queue, execute a SQL insert, etc.

@Bean
public IntegrationFlow fileWritingFlow() {
    return IntegrationFlow.from("outputChannel")
        // Outbound Adapter: Writes the payload to a file
        .handle(Files.outboundAdapter(new File("/tmp/wms/outbound"))
                     .fileExistsMode(FileExistsMode.APPEND)
                     .appendNewLine(true))
        .get();
}

3. The Factory Classes

Notice how we used Files.inboundAdapter() and Files.outboundAdapter()? Spring Integration provides factory classes for almost every technology: Amqp, Http, Jdbc, Mail, Ftp, Kafka, etc. You just need to include the specific Spring Integration module dependency in your pom.xml (e.g., spring-integration-file).
