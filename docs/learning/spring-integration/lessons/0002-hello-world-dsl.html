Lesson 2: Hello World with Java DSL

Since you are comfortable with basic Spring concepts, we are going to skip the legacy XML configuration and jump straight into the modern way to build Spring Integration applications: the Java DSL (Domain Specific Language).

The Java DSL allows you to build complex integration flows using a fluent builder API within a standard Spring @Configuration class.

The IntegrationFlow

The core interface you'll use is IntegrationFlow. You declare it as a Spring Bean. Let's look at a simple example that takes a string, converts it to uppercase, and prints it.

import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.integration.dsl.IntegrationFlow;

@Configuration
public class BasicFlowConfig {

    @Bean
    public IntegrationFlow helloWorldFlow() {
        return IntegrationFlow
                // 1. Declare the input channel
                .from("helloChannel") 
                
                // 2. Add an Endpoint (a Transformer in this case)
                .<String, String>transform(String::toUpperCase) 
                
                // 3. Add an Endpoint (a Service Activator to print)
                .handle(message -> System.out.println("Result: " + message.getPayload())) 
                
                // 4. Build the flow
                .get(); 
    }
}

What is happening here?

  1. from("helloChannel"): We tell the flow to listen to a channel named "helloChannel". Spring Integration will automatically create a DirectChannel bean with this name for you if it doesn't exist.
  2. transform(...): This is an Endpoint. It takes the message payload off the channel, applies a function (converting to uppercase), and implicitly puts the new message onto a hidden, automatically generated channel to pass to the next step.
  3. handle(...): This is the final Endpoint (a Service Activator). It consumes the message and performs an action (printing it). Because it doesn't return anything, the flow ends here.
  4. get(): Finalizes the builder and returns the IntegrationFlow object.
Implicit Channels: Notice we only explicitly named the input channel. The DSL automatically wires up hidden channels between transform() and handle(). You only need to explicitly name channels when you want to reference them from outside the flow.
