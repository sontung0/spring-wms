# User Preferences & Notes

- **Lesson Assessments:** The user prefers the "Action" questions at the end of each lesson to be comprehensive. Instead of asking a single question about one concept, the assessment should test *all* the key knowledge points covered in that specific lesson to ensure thorough retrieval practice.

- **Strict Sequential Progression:** To avoid skipping topics, the teacher must explicitly read `RESOURCES.md` before creating any new lesson. The teacher must identify the last completed topic (by checking the latest `learning-records`) and select the *very next* uncompleted topic in the list. Do not group or skip topics unless explicitly requested by the user.
