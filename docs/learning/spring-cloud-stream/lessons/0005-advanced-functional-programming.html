Advanced Functional Programming

Composition, Reactive Streams, and Multiple Functions.

1. Functional Composition

You can dynamically compose complex handlers from a set of simple functions using the pipe symbol (|). This is incredibly useful for separating cross-cutting concerns (like enriching headers) from business logic.

@Bean
public Function<String, String> uppercase() { return String::toUpperCase; }

@Bean
public Function<String, String> wrapInQuotes() { return s -> "\"" + s + "\""; }

You can compose them via configuration without changing Java code:

spring.cloud.function.definition=uppercase|wrapInQuotes

The resulting binding names will be long (e.g., uppercase|wrapInQuotes-in-0), but you can rename them using spring.cloud.stream.function.bindings.uppercase|wrapInQuotes-in-0=myInput.

2. Multiple Functions in One App

If you have multiple functions in your application, Spring Cloud Stream won't know which one(s) to bind to the broker. You must explicitly tell it using a semicolon (;).

spring.cloud.function.definition=uppercase;reverse

This will create bindings for both functions independently: uppercase-in-0 and reverse-in-0.

3. Reactive Functions

Spring Cloud Function is built on top of Project Reactor. You can write your functions using Flux and Mono to process streams of data reactively.

@Bean
public Function<Flux<String>, Flux<String>> reactiveUpperCase() {
    return flux -> flux.map(String::toUpperCase);
}
Warning: Reactive features like back-pressure only truly work end-to-end if you are using a reactive binder (like the Reactive Kafka Binder). If you use the standard Kafka binder, you get the convenience of the Flux API, but not the underlying reactive back-pressure guarantees.

4. Batch Consumers

Sometimes you want to process messages in batches rather than one by one. If your binder supports it (like Kafka), you can enable batch mode and change your function signature to accept a List.

spring.cloud.stream.bindings.myFunction-in-0.consumer.batch-mode=true
@Bean
public Function<List<Person>, Person> findFirstPerson() {
    return persons -> persons.get(0);
}
