Lesson 4: Multiple Functions and Routing

So far, we've assumed an application only has a single Function or Consumer bean. But real-world microservices often handle multiple different message streams.

Multiple Functions in One App

If you define more than one functional bean in your configuration, Spring Cloud Stream doesn't know which ones it should bind to the broker by default. You have to tell it explicitly using the spring.cloud.function.definition property.

@Configuration
public class EventProcessors {

    @Bean
    public Consumer<Order> processOrder() { ... }

    @Bean
    public Consumer<Invoice> processInvoice() { ... }
}

In your application.yml, you use a semicolon-separated list to activate them:

spring:
  cloud:
    function:
      definition: processOrder;processInvoice
    stream:
      bindings:
        processOrder-in-0:
          destination: orders-topic
        processInvoice-in-0:
          destination: invoices-topic

StreamBridge: Sending Messages Dynamically

The Function interface is great if you have exactly one input going to exactly one output. But what if you have a REST endpoint that needs to send a message? Or what if you process an order and need to send it to either the "approved" topic or the "rejected" topic based on some business logic?

For imperative, dynamic sending, Spring Cloud Stream provides StreamBridge.

You can autowire StreamBridge anywhere in your app and use it to send data to a binding name (or directly to a destination).

import org.springframework.cloud.stream.function.StreamBridge;
import org.springframework.web.bind.annotation.*;

@RestController
public class OrderController {

    private final StreamBridge streamBridge;

    public OrderController(StreamBridge streamBridge) {
        this.streamBridge = streamBridge;
    }

    @PostMapping("/orders")
    public String createOrder(@RequestBody Order order) {
        
        if (order.isValid()) {
            // Send to a binding named 'approvedOrders-out-0'
            streamBridge.send("approvedOrders-out-0", order);
        } else {
            // Send to a binding named 'rejectedOrders-out-0'
            streamBridge.send("rejectedOrders-out-0", order);
        }
        
        return "Order submitted for processing";
    }
}
