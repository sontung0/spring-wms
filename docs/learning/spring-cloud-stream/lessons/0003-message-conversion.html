Lesson 3: Message Conversion & Headers

Brokers like Kafka and RabbitMQ do not know about your Java objects. To them, a message is just a stream of raw bytes. So, how does Spring Cloud Stream magically turn those bytes into a complex OrderEvent object?

Automatic Message Conversion

Spring Cloud Stream handles serialization (object to bytes) and deserialization (bytes to object) automatically using a concept called Message Converters.

By default, Spring assumes your data is JSON (application/json). If you define a Function<OrderEvent, Receipt>, Spring will use Jackson (the JSON library) to parse the incoming byte array into an OrderEvent, and then serialize your returned Receipt back into JSON bytes before sending it to the broker.

Explicit Content-Type Configuration

Usually, you don't have to configure anything. But if you need to be explicit (or if you are using Avro, XML, etc.), you configure it per binding:

spring:
  cloud:
    stream:
      bindings:
        processOrder-in-0:
          destination: incoming-orders
          content-type: application/json

Payloads vs. Full Messages (Headers)

Up until now, our functions just took the domain object directly (the Payload):

@Bean
public Consumer<OrderEvent> processOrder() { ... }

But what if you need to read Headers (metadata) from Kafka or RabbitMQ? (e.g., a tracing ID, a timestamp, or a custom header the sender attached).

Instead of receiving just the payload, you can ask Spring to give you the entire org.springframework.messaging.Message<T> wrapper:

import org.springframework.messaging.Message;
import java.util.function.Consumer;

@Bean
public Consumer<Message<OrderEvent>> processOrder() {
    return message -> {
        // 1. Get the payload
        OrderEvent order = message.getPayload();
        
        // 2. Get the headers
        String correlationId = (String) message.getHeaders().get("x-correlation-id");
        
        System.out.println("Processing order with ID: " + correlationId);
    };
}

Knowledge Check

You have written a function that calculates a discount and needs to return an Order object, but you also need to attach a custom header named discount-applied: true to the outgoing message.

What should the return type of your Function be?

  1. Order
  2. Message<Order>
  3. MessageHeaders
Correct! You wrap your output object in a Message using MessageBuilder.withPayload(order).setHeader("discount-applied", true).build().
Not quite. If you just return Order, Spring will only send the payload and you won't be able to attach headers.
