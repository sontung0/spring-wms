Event Routing

Dynamically routing messages to functions, or routing outputs to dynamic destinations.

1. Routing TO a Consumer (Inbound Routing)

Sometimes you have a single input topic, but you want to process the messages differently based on their content. For example, you might want to route "VIP" orders to a processVipOrder function, and standard orders to a processStandardOrder function.

To do this, you enable the built-in RoutingFunction:

spring.cloud.stream.function.routing.enabled=true

When enabled, the framework exposes a special input binding named functionRouter-in-0. You bind this to your topic:

spring.cloud.stream.bindings.functionRouter-in-0.destination=orders-topic

How does it know which function to call?

Option A: SpEL Expressions
You can provide a SpEL expression via application properties to evaluate the message (e.g., checking a header):

spring.cloud.function.routing-expression=headers['orderType'] == 'VIP' ? 'processVipOrder' : 'processStandardOrder'

Option B: MessageRoutingCallback
For complex logic, you can write a Java bean that implements MessageRoutingCallback. This takes precedence over SpEL expressions.

@Bean
public MessageRoutingCallback customRouter() {
    return message -> {
        Order payload = (Order) message.getPayload();
        return payload.getAmount() > 1000 ? "processVipOrder" : "processStandardOrder";
    };
}

2. Routing FROM a Consumer (Outbound Routing)

What if your function processes a message, and you want to dynamically decide which topic to send the result to?

You can do this by returning a Message object and setting a special header: spring.cloud.stream.sendto.destination.

@Bean
public Function<Order, Message<Order>> processOrder() {
    return order -> {
        String targetTopic = order.isInternational() ? "intl-orders" : "domestic-orders";
        
        return MessageBuilder.withPayload(order)
            .setHeader("spring.cloud.stream.sendto.destination", targetTopic)
            .build();
    };
}

The framework will intercept this header, ignore the standard output binding, and dynamically send the message to the topic specified in the header.

Note: You can also use StreamBridge (which we learned about in Lesson 4) to send messages to dynamic destinations programmatically!
