Explicit Bindings, Lifecycle Control, and Post-Processing

Wrapping up the Programming Model with advanced binding management.

1. Explicit Binding Creation

We know that bindings are created implicitly based on your Function names (e.g., myFunction-in-0). But what if you want to create a binding that isn't tied to a function? This is common when you want to use StreamBridge to send messages to a pre-configured destination.

You can create bindings explicitly using application properties:

spring.cloud.stream.input-bindings=myInput;anotherInput
spring.cloud.stream.output-bindings=myOutput;anotherOutput
Important: Explicit bindings do not get the -in-0 or -out-0 suffix. The binding name is exactly what you typed. You configure it like this: spring.cloud.stream.bindings.myOutput.destination=my-topic.

2. Binding Visualization and Control

Sometimes you need to stop consuming messages temporarily (e.g., during maintenance) without shutting down the whole application. Spring Cloud Stream allows you to control the lifecycle of individual bindings at runtime.

Via Actuator (HTTP)

If you include spring-boot-starter-actuator and expose the bindings endpoint, you can view all bindings at /actuator/bindings.

You can stop a binding by sending a POST request:

curl -d '{"state":"STOPPED"}' -H "Content-Type: application/json" -X POST http://localhost:8080/actuator/bindings/myFunction-in-0

Via Java (Programmatically)

You can inject the BindingsLifecycleController bean to do the same thing in code:

@Autowired
private BindingsLifecycleController bindingsController;

public void pauseConsumption() {
    bindingsController.changeState("myFunction-in-0", State.STOPPED);
}

3. Post-Processing (After Sending)

Normally, when your function returns a value, the framework sends it to the broker. But what if you need to execute some logic strictly after the message has been successfully sent to the broker (e.g., updating a database to mark a record as "published")?

You can implement the PostProcessingFunction interface, which adds a postProcess(Message<O> result) method to your function.

public class MyProcessor implements PostProcessingFunction<String, String> {
    @Override
    public String apply(String input) {
        return input.toUpperCase();
    }

    @Override
    public void postProcess(Message<String> result) {
        System.out.println("Message successfully sent to the broker!");
    }
}
