Spring Cloud Stream Glossary

Source / Supplier
An application that publishes messages. Represented by java.util.function.Supplier.
Processor / Function
An application that consumes messages, transforms them, and publishes the result. Represented by java.util.function.Function.
Sink / Consumer
An application that consumes messages and does not publish any outputs back to the messaging system. Represented by java.util.function.Consumer.
Binder
The component responsible for providing integration with the physical messaging system (e.g., Kafka, RabbitMQ).
Binding
The logical bridge between your code (the Java function) and a physical destination (topic/queue) on the broker.
Consumer Group
A mechanism to allow multiple instances of an application to scale out message consumption. Each message is processed by only one instance in the group.
MessageConverter
A Spring framework mechanism that automatically converts incoming byte arrays from the broker into Java objects, and outgoing Java objects into byte arrays.
Message<T>
A Spring messaging wrapper object that contains both the payload (your data) and headers (metadata about the message).
StreamBridge
A utility class provided by Spring Cloud Stream used to imperatively send messages to output bindings or destinations at runtime (e.g., from a REST controller or conditionally within a Consumer).
DLQ (Dead Letter Queue)
A secondary queue or topic where messages are sent if they cannot be processed successfully by the application (usually after retries are exhausted). Essential for preventing data loss.
TestBinder
An in-memory, simulated binder provided by spring-cloud-stream-test-binder used for fast, isolated unit/integration testing.
InputDestination / OutputDestination
Testing utilities used with the TestBinder to inject messages into and read messages out of your application's bindings during a test.
